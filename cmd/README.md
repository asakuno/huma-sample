# Command ディレクトリ

## 役割・責務
このディレクトリは、アプリケーションのエントリーポイント（main関数）を含むコマンド群を管理します。

## 構造
- `server/` - HTTPサーバーのメイン実行ファイル
- `migration/` - データベースマイグレーション実行コマンド

## 設計原則
- main関数は最小限に保ち、実際のロジックはapp層に委譲
- 各コマンドは独立して実行可能
- 設定の読み込みとDI（依存注入）の初期化を担当

## AI開発時の注意点
- このディレクトリのファイルはアプリケーションの起動設定のみ
- ビジネスロジックは含めず、app層のサービスを呼び出すだけ
- エラーハンドリングは最低限、詳細はapp層で処理
